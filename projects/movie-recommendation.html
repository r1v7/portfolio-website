Movie Recommendation System

Project Overview

This project aimed to develop a personalized movie recommendation system using machine learning to address the challenge of content overload on streaming platforms. By leveraging collaborative filtering (user ratings) and content-based filtering (metadata like genres, descriptions), the team built a hybrid model to deliver accurate and diverse recommendations. The system utilized a Kaggle dataset of 129,890 entries, focusing on enhancing user experience through tailored suggestions.

Design Process

Data Collection & Preprocessing: Sourced metadata (genres, ratings, cast, etc.) from Kaggle. Cleaned missing values (e.g., replaced missing runtime/director fields with "Unknown") and standardized formats. Processed text data (descriptions, cast) using TF-IDF vectorization.

Exploratory Data Analysis (EDA): Analyzed rating distributions, genre frequency, and correlations (e.g., votes vs. ratings). Identified biases: genre imbalance (drama/comedy dominance) and popularity bias (high-rated movies receiving more votes).

Model Development: Applied Singular Value Decomposition (SVD) using the Surprise library for collaborative filtering (RMSE: 0.28, MAE: 0.22). Used TF-IDF to vectorize textual features and compute cosine similarity for content-based filtering. Combined both approaches via weighted averaging to balance accuracy and diversity in the hybrid model.

Evaluation: Tested models using RMSE, MAE, and recommendation accuracy metrics. Visualized results (e.g., accuracy comparisons, top recommendations).

Key Features

  • Hybrid Approach: Merged collaborative and content-based filtering to improve recommendation diversity and precision
  • Metadata Utilization: Leveraged genres, cast, descriptions, and ratings for personalized suggestions
  • Bias Mitigation: Addressed genre imbalance and popularity bias through preprocessing and hybrid scoring
  • Scalability: Implemented PySpark and AWS EMR for handling large datasets

Technologies Used

  • Python for core development
  • Pandas, NumPy for preprocessing
  • Matplotlib, Seaborn for visualization
  • Scikit-learn, Surprise for ML models
  • PySpark for distributed computing
  • AWS EMR for cloud-based processing
  • Jupyter Notebook for prototyping

Challenges & Solutions

  • Data Sparsity: Imputed missing data with "Unknown" or predictive models
  • Popularity Bias: Adjusted hybrid model weights to balance popularity and relevance
  • Genre Imbalance: Implemented custom taxonomies and feature reweighting
  • Real-Time Processing: Leveraged PySpark and AWS EMR for distributed computing

Outcomes & Learnings

  • Hybrid model achieved 82.71% weighted accuracy
  • Collaborative filtering showed low error rates (RMSE: 0.28)
  • Content-based filtering achieved 82% average recommendation accuracy
  • Learned that hybrid models effectively mitigate individual method limitations
  • Discovered importance of data preprocessing for model reliability
  • Identified need for proactive bias mitigation strategies
